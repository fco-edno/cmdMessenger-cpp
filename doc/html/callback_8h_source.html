29 /*
30 To use:
31 
32 If you wish to build a component that provides/needs a callback, simply
33 specify and hold a CBFunctor of the type corresponding to the args
34 you wish to pass and the return value you need. There are 10 Functors
35 from which to choose:
36 
37 CBFunctor0
38 CBFunctor1<P1>
39 CBFunctor2<P1,P2>
40 CBFunctor3<P1,P2,P3>
41 CBFunctor4<P1,P2,P3,P4>
42 CBFunctor0wRet<RT>
43 CBFunctor1wRet<P1,RT>
44 CBFunctor2wRet<P1,P2,RT>
45 CBFunctor3wRet<P1,P2,P3,RT>
46 CBFunctor4wRet<P1,P2,P3,P4,RT>
47 
48 These are parameterized by their args and return value if any. Each has
49 a default ctor and an operator() with the corresponding signature.
50 
51 They can be treated and used just like ptr-to-functions.
52 
53 If you want to be a client of a component that uses callbacks, you
54 create a CBFunctor by calling makeFunctor().
55 
56 There are three flavors of makeFunctor - you can create a functor from:
57 
58 a ptr-to-stand-alone function
59 an object and a pointer-to-member function.
60 a pointer-to-member function (which will be called on first arg of functor)
61 
62 The current iteration of this library requires you pass makeFunctor()
63 a dummy first argument of type ptr-to-the-Functor-type-you-want-to-create.
64 Simply cast 0 to provide this argument:
65 
66 makeFunctor((target-functor*)0,ptr-to-function)
67 makeFunctor((target-functor*)0,reference-to-object,ptr-to-member-function)
68 makeFunctor((target-functor*)0,ptr-to-member-function)
69 
70 Future versions will drop this requirement once member templates are
71 available.
72 
73 The functor system is 100% type safe. It is also type flexible. You can
74 build a functor out of a function that is 'type compatible' with the
75 target functor - it need not have an exactly matching signature. By
76 type compatible I mean a function with the same number of arguments, of
77 types reachable from the functor's argument types by implicit conversion.
78 The return type of the function must be implicitly convertible to the
79 return type of the functor. A functor with no return can be built from a
80 function with a return - the return value is simply ignored.
81 (See ethel example below)
82 
83 All the correct virtual function behavior is preserved. (see ricky
84 example below).
85 
86 If you somehow try to create something in violation
87 of the type system you will get a compile-time or template-instantiation-
88 time error.
89 
90 The CBFunctor base class and translator
91 classes are artifacts of this implementation. You should not write
92 code that relies upon their existence. Nor should you rely on the return
93 value of makeFunctor being anything in particular.
94 
95 All that is guaranteed is that the Functor classes have a default ctor,
96 an operator() with the requested argument types and return type, an
97 operator that will allow it to be evaluated in a conditional (with
98 'true' meaning the functor is set, 'false' meaning it is not), and that
99 Functors can be constructed from the result of makeFunctor(), given
100 you've passed something compatible to makeFunctor().
